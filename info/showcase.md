# Showcase - Laboratorio de Componentes

## Propósito

La ruta `/showcase` funciona como un laboratorio de pruebas para visualizar y comparar diferentes variantes de componentes UI antes de implementarlos en producción.

## Estructura

### Organización por Secciones

Cada sección del sitio tiene su propio apartado con múltiples variantes:

```
/showcase
├── Sección Hero
│   ├── Variante 1
│   ├── Variante 2
│   └── Variante 3
├── Sección Servicios
│   ├── Variante 1
│   ├── Variante 2
│   └── Variante 3
├── Sección Testimonios
│   ├── Variante 1
│   └── Variante 2
└── ...
```

## Directrices de Diseño

### Paleta de Colores

- Seguir la paleta de colores establecida en el proyecto
- Experimentar con diferentes combinaciones dentro de la paleta
- Mantener consistencia visual entre variantes

### Estados e Interactividad

Cada componente debe incluir:

- **Hover**: Efecto al pasar el cursor
- **Active**: Estado al hacer clic
- **Disabled**: Estado deshabilitado (si aplica)
- **Focus**: Estado de enfoque para accesibilidad

### Mejoras Sugeridas

Cada variante puede incluir:

- Animaciones con Framer Motion
- Transiciones suaves
- Efectos visuales experimentales
- Optimizaciones de UX

## Uso

1. Navegar a `/showcase`
2. Revisar las diferentes variantes de cada sección
3. Comparar estilos y comportamientos
4. Seleccionar la variante preferida para implementar

## Notas

- Este es un entorno de pruebas, no de producción
- Las variantes pueden ser modificadas o eliminadas
- Usar como referencia para decisiones de diseño
